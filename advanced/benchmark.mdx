## Overview

`profile_inference.py` is a comprehensive profiling and benchmarking tool for ACE-Step 1.5. It measures end-to-end wall time, LLM planning time, DiT diffusion time, VAE decoding time, and more across different devices, backends, and configurations.

## Quick Start

```bash
# Basic profile (text2music, default settings)
python profile_inference.py

# Profile with LLM thinking enabled
python profile_inference.py --thinking

# Run benchmark matrix
python profile_inference.py --mode benchmark

# Profile on Apple Silicon
python profile_inference.py --device mps --lm-backend mlx

# Profile with cProfile function-level analysis
python profile_inference.py --detailed
```

## Profiling Modes

### Profile Mode (Single Run)

Runs a single generation with detailed timing breakdown. Includes optional warmup and cProfile.

```bash
python profile_inference.py --mode profile
```

**What it measures:**
- Total wall time (end-to-end)
- LLM planning time (token generation, constrained decoding, CFG overhead)
- DiT diffusion time (per-step and total)
- VAE decode time
- Audio save time

**Key options:**

| Flag | Description |
|------|-------------|
| `--no-warmup` | Skip warmup run (includes compilation overhead) |
| `--detailed` | Enable `cProfile` function-level analysis |
| `--llm-debug` | Deep LLM debugging (token count, throughput) |
| `--thinking` | Enable LLM Chain-of-Thought reasoning |
| `--duration <sec>` | Override audio duration |
| `--batch-size <n>` | Override batch size |
| `--inference-steps <n>` | Override diffusion steps |

### Benchmark Mode (Configuration Matrix)

Runs a matrix of configurations and produces a summary table. Automatically adapts to GPU memory limits.

```bash
python profile_inference.py --mode benchmark
```

**Default matrix:**
- Durations: 30s, 60s, 120s, 240s (clamped by GPU)
- Batch sizes: 1, 2, 4 (clamped by GPU)
- Thinking: True, False
- Inference steps: 8, 16

**Example output:**

```
Duration   Batch   Think   Steps   Wall(s)    LM(s)      DiT(s)     VAE(s)     Status
------------------------------------------------------------------------------------------
30         1       False   8       3.21       0.45       1.89       0.52       OK
30         1       True    8       5.67       2.91       1.89       0.52       OK
60         2       False   16      12.34      0.48       9.12       1.85       OK
```

**Save results to JSON:**

```bash
python profile_inference.py --mode benchmark --benchmark-output results.json
```

### Understand Mode (Audio Understanding)

Profiles the `understand_music()` API which extracts metadata (BPM, key, time signature, caption) from audio codes.

```bash
python profile_inference.py --mode understand
python profile_inference.py --mode understand --audio-codes "<|audio_code_123|>..."
```

### Create Sample Mode (Inspiration)

Profiles the `create_sample()` API which generates a complete song blueprint from a text query.

```bash
python profile_inference.py --mode create_sample
python profile_inference.py --mode create_sample --sample-query "a soft Bengali love song"
python profile_inference.py --mode create_sample --instrumental
```

### Format Sample Mode (Metadata)

Profiles the `format_sample()` API which converts caption + lyrics into structured metadata.

```bash
python profile_inference.py --mode format_sample
```

## Device & Backend Selection

### Supported Devices

| Device | Flag | Notes |
|--------|------|-------|
| CUDA (NVIDIA) | `--device cuda` | Auto-detected, recommended |
| MPS (Apple Silicon) | `--device mps` | macOS M-series chips |
| CPU | `--device cpu` | Slow, testing only |
| Auto | `--device auto` | Selects best available (default) |

### Supported LLM Backends

| Backend | Flag | Notes |
|---------|------|-------|
| vLLM | `--lm-backend vllm` | Fastest on NVIDIA |
| PyTorch | `--lm-backend pt` | Universal fallback |
| MLX | `--lm-backend mlx` | Optimized for Apple Silicon |
| Auto | `--lm-backend auto` | Selects best backend (default) |

### Examples

```bash
# NVIDIA GPU
python profile_inference.py --device cuda --lm-backend vllm

# Apple Silicon
python profile_inference.py --device mps --lm-backend mlx

# CPU baseline
python profile_inference.py --device cpu --lm-backend pt
```

The tool automatically identifies bottlenecks:

**LLM bottleneck:**
```
LLM is the bottleneck: 2.9s (45% of total)
Suggestions:
  1. Run with --llm-debug for token-level throughput analysis
  2. Try --no-constrained-decoding to reduce FSM overhead
  3. Compare backends: --lm-backend vllm vs pt vs mlx
  4. Reduce lm_cfg_scale (currently doubles forward passes if > 1.0)
```

```
DiT is the bottleneck: 9.1s (75% of total)
Suggestions:
  1. Reduce --inference-steps (turbo model supports 4-8)
  2. Reduce --duration
  3. Try --quantization int8_weight_only
```

## CLI Reference

### Device & Backend

```bash
--device auto|cuda|mps|cpu          # Device selection (default: auto)
--lm-backend auto|vllm|pt|mlx       # LLM backend (default: auto)
```

### Model Paths

```bash
--config-path acestep-v15-turbo     # DiT model config
--lm-model acestep-5Hz-lm-1.7B      # LLM model path
```

### Hardware Options

```bash
--offload-to-cpu                    # Offload models to CPU when idle
--offload-dit-to-cpu                # Offload DiT to CPU when idle
--quantization int8_weight_only     # Quantization mode
```

### Generation Parameters

```bash
--duration 60                       # Audio duration (seconds)
--batch-size 2                      # Batch size
--inference-steps 8                 # Diffusion steps
--seed 42                           # Random seed
--guidance-scale 7.0                # CFG guidance scale
```

### LLM / CoT Parameters

```bash
--thinking                          # Enable LLM reasoning
--use-cot-metas                     # LLM generates metadata
--use-cot-caption                   # LLM rewrites caption
--use-cot-language                  # LLM detects language
--use-constrained-decoding          # FSM-based decoding (default)
--no-constrained-decoding           # Disable constrained decoding
--lm-temperature 0.85               # LLM sampling temperature
--lm-cfg-scale 2.0                  # LLM CFG scale
```

### Profiling Options

```bash
--mode profile                      # Single run profiling (default)
--mode benchmark                    # Configuration matrix
--mode understand                   # Profile understand_music()
--mode create_sample                # Profile create_sample()
--mode format_sample                # Profile format_sample()
--no-warmup                         # Skip warmup run
--detailed                          # Enable cProfile analysis
--llm-debug                         # Deep LLM debugging
--benchmark-output results.json     # Save results to JSON
```

### Compare LLM Models


### Thinking vs No-Thinking

```bash

  --offload-to-cpu \
  --quantization int8_weight_only \

```

```bash
# Run full benchmark and save results

  --mode benchmark \
  --benchmark-output benchmark_results.json

cat benchmark_results.json | python -m json.tool
```

### Function-Level Profiling

```bash
# Enable cProfile for detailed analysis
python profile_inference.py --detailed --llm-debug
```

## Tips & Best Practices

1. **Always include warmup** (default) — The first run includes JIT compilation and memory allocation overhead. Warmup ensures measurements reflect steady-state performance.

2. **Use `--benchmark-output`** to save results as JSON for later analysis or comparison across hardware.

3. **Compare with thinking off vs on** — Thinking mode significantly increases LLM time but may improve generation quality.

4. **Test with representative durations** — Short durations (30s) are dominated by LLM time; long durations (240s+) are dominated by DiT time.

5. **GPU memory auto-adaptation** — Benchmark mode automatically clamps durations and batch sizes based on detected GPU tier.

6. **Use `--detailed` sparingly** — `cProfile` adds overhead; use it only when investigating function-level bottlenecks.
